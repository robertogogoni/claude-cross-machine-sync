# Code Explanation

Explain the code in a clear, educational manner:

1. **High-Level Overview**
   - What does this code do?
   - What problem does it solve?
   - Use a real-world analogy

2. **Visual Diagram**
   - Draw ASCII art showing the flow or architecture
   - Illustrate relationships between components
   - Show data flow or state changes

3. **Step-by-Step Walkthrough**
   - Explain the code line-by-line or block-by-block
   - Highlight key logic and decision points
   - Explain any complex algorithms or patterns

4. **Key Concepts**
   - Important variables and their purpose
   - Design patterns used
   - Edge cases and how they're handled

5. **Common Pitfalls**
   - What could go wrong?
   - Common mistakes when working with this code
   - Things to watch out for

Keep explanations conversational and accessible. Use multiple analogies for complex concepts.
