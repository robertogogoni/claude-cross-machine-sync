# Chrome Native Messaging for Non-Standard Channels

**Created**: 2026-03-18
**Problem**: Claude Code only auto-registers native messaging hosts for Chrome and Chromium, not Canary

## How Claude in Chrome works

```
Chrome extension (fcoeoabgfenejglbffodgkkbkcdhcgfn)
    |  chrome.runtime.connectNative()
    v
Native messaging host (com.anthropic.claude_code_browser_extension)
    |  reads manifest from NativeMessagingHosts/
    v
~/.claude/chrome/chrome-native-host (shell script)
    |  exec claude-binary --chrome-native-host
    v
Unix socket at /tmp/claude-mcp-browser-bridge-<user>/<pid>.sock
    ^
    |  Claude Code CLI connects as client
Claude Code session
```

## The native host script has a hardcoded version path
```sh
exec "/home/user/.local/share/claude/versions/2.1.78" --chrome-native-host
```
Claude Code regenerates this script on self-update (comment says "Generated by Claude Code").

## Fix for Chrome Canary
Symlink instead of copy:
```
~/.config/google-chrome-canary/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json
  -> ~/.config/google-chrome/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json
```

Why symlink: Claude Code updates the Chrome manifest. Canary gets the update via symlink. No maintenance.

## Directories per Chrome channel
- `~/.config/google-chrome/NativeMessagingHosts/` (auto-managed by Claude Code)
- `~/.config/chromium/NativeMessagingHosts/` (auto-managed by Claude Code)
- `~/.config/google-chrome-canary/NativeMessagingHosts/` (manual, use symlink)
- `~/.config/google-chrome-beta/NativeMessagingHosts/` (manual, use symlink)
- `~/.config/google-chrome-unstable/NativeMessagingHosts/` (manual, use symlink)

## Known limitation
The MCP bridge initializes at Claude Code CLI session start. If the socket doesn't exist yet (Chrome not running or extension not activated), it caches "not connected" and won't retry. Must restart the CLI session.

## Service worker activation
MV3 extension service workers are lazy. After Chrome restart, the service worker needs a trigger to call connectNative():
- `runtime.onStartup` event (fires on Chrome launch)
- `action.onClicked` (user clicks extension icon)
- Content script match on claude.ai pages
If the native host was missing on first startup, restarting Chrome after adding the manifest is required.
