Investigated Claude Code's memory system, which consists of 5 hierarchical levels (enterprise policy, project, project rules, user, and local project memory) stored as markdown files in CLAUDE.md and .claude/rules/ directories. Memory files are loaded recursively from the current directory upward, support file imports with @ syntax, and can include path-specific rules with glob patterns via YAML frontmatter. The memory system is text-based and file-driven—there is no programmatic API available; interaction occurs through the /memory slash command or direct file editing.
