When working on a project deliverable (like a GitHub repo, docs, installer), user wants full creative and technical autonomy. Don't ask permission for every change. Alternate between building and quality-checking. Add missing ingredients, fix problems, expand weak sections, trim bloat, all without stopping to ask.

**Why:** User is the head chef setting direction; Claude is the sous-chef executing with taste and judgment. Stopping to ask "should I add salt?" breaks the flow. Just add it if it needs salt.

**How to apply:** On creative/deliverable work, iterate freely: build, taste-test (review), adjust, build more. Only check in at natural milestones or when direction needs to change. For destructive system operations, still confirm.
